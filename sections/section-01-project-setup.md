## Background

**Cortex 2.0** is an Obsidian plugin that replaces a plain-text inbox file (`_Cortex.md`) with an interactive checklist UI. Items are standard markdown checkboxes (`- [ ] text ![[image]]`). The plugin is cross-platform: desktop (Windows, macOS) and mobile (Android).

**Key architectural decisions that affect the build setup:**

- Plugin must work on mobile (`isDesktopOnly: false`)
- Testing framework is **vitest** for pure logic modules (no Obsidian API involvement)
- Bundler is **esbuild** (standard for Obsidian plugins)
- No `moment.js` dependency — timestamps use native `Date`
- `obsidian` is an external (peer) dependency, not bundled
- Minimum Obsidian app version `1.5.7` — required for `fileManager.getAvailablePathForAttachment()`

Note: `obsidian` is in `devDependencies` because it provides type definitions only — the actual Obsidian API is available at runtime inside Obsidian itself and must not be bundled.

### `tsconfig.json`

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "inlineSourceMap": true,
    "inlineSources": true,
    "module": "ESNext",
    "target": "ES2018",
    "allowImportingTsExtensions": false,
    "moduleResolution": "bundler",
    "importHelpers": true,
    "isolatedModules": true,
    "strictNullChecks": true,
    "lib": ["ES2018", "DOM"]
  },
  "include": ["**/*.ts"],
  "exclude": ["node_modules"]
}
```

`target: ES2018` matches the esbuild target. `strictNullChecks: true` catches null-safety issues early. `lib: ["ES2018", "DOM"]` provides browser DOM types needed for event handlers.

### `esbuild.config.mjs`

Standard Obsidian plugin esbuild setup:

```javascript
import esbuild from "esbuild";
import process from "process";
import builtins from "builtin-modules";

const banner = `/*
THIS IS A GENERATED/BUNDLED FILE BY ESBUILD
if you want to view the source, please visit the github repository of this plugin
*/
`;

const prod = process.argv[2] === "production";

const context = await esbuild.context({
  banner: {
    js: banner,
  },
  entryPoints: ["main.ts"],
  bundle: true,
  external: [
    "obsidian",
    "electron",
    "@codemirror/autocomplete",
    "@codemirror/collab",
    "@codemirror/commands",
    "@codemirror/language",
    "@codemirror/lint",
    "@codemirror/search",
    "@codemirror/state",
    "@codemirror/view",
    "@lezer/common",
    "@lezer/highlight",
    "@lezer/lr",
    ...builtins,
  ],
  format: "cjs",
  target: "es2018",
  logLevel: "info",
  sourcemap: prod ? false : "inline",
  treeShaking: true,
  outfile: "main.js",
});

if (prod) {
  await context.rebuild();
  process.exit(0);
} else {
  await context.watch();
}
```

This requires the `builtin-modules` package. Add it to `devDependencies`:

```json
"builtin-modules": "^3.3.0"
```

### `vitest.config.ts`

```typescript
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    environment: "node",
    include: ["src/**/*.test.ts"],
  },
});
```

Tests use `node` environment (not jsdom) because `CortexFile.ts` is pure logic with no DOM dependency. If DOM tests are needed later for `CortexView`, add a separate config or use `// @vitest-environment jsdom` inline.
